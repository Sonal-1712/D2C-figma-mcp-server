# Workflows

User can trigger one of your workflows, by providing workflow name in following format: 

```
§<entity>:<action>
```

List of all available workflows are kept under following relative folder `../workflows/*`.

Example 001

```
§components:generate
```

Above means -> you have to read following file: 

design-2-code-v1-code-agent/workflows/components-generate.md

And proceed accordingly.

Example 002

```
§screens-generate
```

Above means -> you have to read following file: 

design-2-code-v1-code-agent/workflows/screens-generate.md

And proceed accordingly.
